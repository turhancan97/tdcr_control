This project provides a Python implementation of modelling and benchmark tools for tendon-driven continuum robots (TDCRs). TDCRs are flexible robots actuated by tendons routed along a compliant backbone; pulling the tendons bends the robot and produces the resulting 3D shape.

The implementation focuses on a two-segment TDCR with six tendons and reproduces the modelling workflow used to compare common TDCR modelling assumptions. It implements:

- `cc`: geometric constant curvature model
- `ccsub`: piecewise constant curvature static model
- `vc`: Cosserat rod model
- `vcref`: subsegment Cosserat rod model
- `prbm`: pseudo-rigid-body model

## Background and Attribution

This work is highly inspired by the paper:

Rao, P., Peyron, Q., Lilge, S., and Burgner-Kahrs, J. (2021). *How to Model Tendon-Driven Continuum Robots and Benchmark Modelling Performance*. Frontiers in Robotics and AI, 7:630245. DOI: [10.3389/frobt.2020.630245](https://doi.org/10.3389/frobt.2020.630245)

The original authors provide MATLAB and C++ implementations with the paper at:

[https://github.com/SvenLilge/tdcr-modeling](https://github.com/SvenLilge/tdcr-modeling)

This repository is an independent Python adaptation of that modelling and benchmarking workflow. It is intended to make the models easier to run, inspect, test, and extend in a Python-based robotics/control workflow.

## Installation

From the repository root:

```bash
python3 -m pip install -e ".[test]"
```

The package depends on `numpy`, `scipy`, `matplotlib`, `plotly`, and `PyYAML`.

## Usage

Run the benchmark with the default config:

```bash
tdcr_benchmark
```

During development, the same command can be run without installation:

```bash
PYTHONPATH=. python3 -m tdcr_benchmark.cli
```

Override tendon tensions:

```bash
tdcr_benchmark --tensions 8 0 0 0 0 0
```

By default, outputs are written to a timestamped directory under `outputs/`. Use `--no-timestamp` for deterministic output paths.

## Config

The default config is `config.yaml`. It defines the two-segment, six-tendon TDCR:

- segment lengths and disk counts
- tendon routing positions
- tendon tensions for static models
- tendon displacements for the geometric constant-curvature model
- backbone material and geometric properties
- gravity and tip loads

The benchmark currently assumes exactly two segments and six tendons.

## Outputs

For each implemented model, the CLI writes:

- `<model>_backbone.csv`: sampled backbone coordinates
- `<model>_tip_pose.csv`: final 4x4 end-effector transform
- `<model>.png`: saved static plot of the backbone when Matplotlib is available
- `<model>.html`: interactive browser plot with rotate/pan/zoom controls, disk rings, and tendon routing lines

## Numerical Notes

The checked-in references are generated by this Python implementation. They are intended for regression testing, not as authoritative MATLAB/C++ parity data. The nonlinear solves use SciPy and may not match MATLAB `fsolve` bit-for-bit.


## Trajectory Animation (v1)

Generate a simple tip-trajectory animation using the `cc` model:

```bash
tdcr_animate --trajectory line --frames 120
```

Circle trajectory example:

```bash
tdcr_animate --trajectory circle --circle-radius 0.03 --plane-z 0.32
```

Animation outputs:

- `animation.html`: interactive frame-by-frame robot animation
- `trajectory_targets.csv`: target tip points
- `trajectory_tip_actual.csv`: actual tip points and per-frame solver status
- `trajectory_tendon_commands.csv`: solved tendon commands (`q1..q6`)
